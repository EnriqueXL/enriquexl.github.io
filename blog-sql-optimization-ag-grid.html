Optimización SQL: Paginado y AG Grid vs DataTables

5 Mar 2024 7 min lectura Enrique Hernández
Performance AG Grid Paginado SQL

La optimización de consultas SQL es crucial para el rendimiento de las aplicaciones. En este artículo comparto mi experiencia implementando paginado del lado del cliente y por qué AG Grid supera por mucho a DataTables de jQuery.

El Problema del Paginado del Lado del Servidor

Tradicionalmente, el paginado se implementa en el servidor, lo que significa que cada cambio de página requiere una nueva consulta SQL. Esto puede ser ineficiente y lento, especialmente con grandes volúmenes de datos.

❌ Paginado del Lado del Servidor

// Consulta SQL con LIMIT y OFFSET
$page = $_GET['page'] ?? 1;
$perPage = 20;
$offset = ($page - 1) * $perPage;

$sql = "SELECT * FROM users 
        ORDER BY created_at DESC 
        LIMIT $perPage OFFSET $offset";

// Cada cambio de página = nueva consulta al servidor
$users = DB::select($sql);

La Solución: Paginado del Lado del Cliente

Implementé una estrategia de paginado del lado del cliente que carga todos los datos una sola vez y maneja la paginación en el navegador. Esto reduce significativamente la carga del servidor y mejora la experiencia del usuario.

✅ Paginado del Lado del Cliente

// Cargar todos los datos una sola vez
public function getAllUsers() {
    return User::select('id', 'name', 'email', 'created_at')
               ->orderBy('created_at', 'desc')
               ->get();
}

// El cliente maneja la paginación
// JavaScript se encarga de mostrar solo los registros necesarios

AG Grid: La Mejor Opción para Tablas de Datos

Después de probar varias librerías, AG Grid se destacó por su rendimiento, funcionalidades avanzadas y flexibilidad. En mi proyecto de sistema de RRHH con Laravel 12, AG Grid demostró ser la elección perfecta para manejar grandes volúmenes de datos de empleados.

Ventajas de AG Grid sobre DataTables

  • Rendimiento: Maneja millones de registros sin problemas
  • Virtual Scrolling: Solo renderiza las filas visibles
  • Funcionalidades Avanzadas: Filtros, ordenamiento, agrupación
  • Personalización: Altamente configurable
  • TypeScript: Soporte nativo para TypeScript
  • React/Angular: Componentes específicos para cada framework

Implementación de AG Grid

AG Grid se integra perfectamente con Laravel y proporciona una experiencia de usuario excepcional.

✅ Configuración de AG Grid

import { GridOptions } from 'ag-grid-community';

const gridOptions: GridOptions = {
  columnDefs: [
    { field: 'id', headerName: 'ID', sortable: true, filter: true },
    { field: 'name', headerName: 'Nombre', sortable: true, filter: true },
    { field: 'email', headerName: 'Email', sortable: true, filter: true },
    { field: 'created_at', headerName: 'Fecha', sortable: true, filter: true }
  ],
  rowData: [], // Se llena con datos del servidor
  pagination: true,
  paginationPageSize: 20,
  domLayout: 'autoHeight',
  defaultColDef: {
    resizable: true,
    sortable: true,
    filter: true
  },
  onGridReady: (params) => {
    // Cargar datos cuando la grilla esté lista
    this.loadData();
  }
};

Integración con Laravel

La integración con Laravel es sencilla y eficiente.

✅ Controlador Laravel

class UserController extends Controller {
    public function index() {
        $users = User::select('id', 'name', 'email', 'created_at')
                    ->orderBy('created_at', 'desc')
                    ->get()
                    ->map(function ($user) {
                        return [
                            'id' => $user->id,
                            'name' => $user->name,
                            'email' => $user->email,
                            'created_at' => $user->created_at->format('Y-m-d H:i:s')
                        ];
                    });

        return response()->json($users);
    }
}

Optimizaciones SQL Adicionales

1. Índices Estratégicos

Los índices correctos son fundamentales para el rendimiento de las consultas.

✅ Índices Recomendados

-- Índice para ordenamiento por fecha
CREATE INDEX idx_users_created_at ON users(created_at DESC);

-- Índice compuesto para búsquedas frecuentes
CREATE INDEX idx_users_name_email ON users(name, email);

-- Índice para filtros comunes
CREATE INDEX idx_users_status ON users(status);

2. Consultas Optimizadas

Evita SELECT * y usa solo las columnas necesarias.

✅ Consulta Optimizada

// ❌ Mal: Selecciona todas las columnas
$users = User::all();

// ✅ Bien: Solo las columnas necesarias
$users = User::select('id', 'name', 'email', 'created_at')
            ->orderBy('created_at', 'desc')
            ->get();

3. Eager Loading

Usa eager loading para evitar el problema N+1.

✅ Eager Loading

// ❌ Mal: Problema N+1
$users = User::all();
foreach ($users as $user) {
    echo $user->profile->name; // Consulta adicional por cada usuario
}

// ✅ Bien: Eager loading
$users = User::with('profile')->get();
foreach ($users as $user) {
    echo $user->profile->name; // Sin consultas adicionales
}

Experiencia Real: Sistema de RRHH con AG Grid

En mi proyecto de sistema de gestión de recursos humanos, implementé AG Grid para manejar datos de empleados, nóminas y reportes. La experiencia fue excepcional, especialmente comparada con DataTables que había usado anteriormente.

Casos de Uso en el Sistema de RRHH

  • Reportes de Nómina: Datos complejos con cálculos y exportación a Excel
  • Dashboard de RRHH: Métricas y KPIs con gráficos integrados

Ventajas Específicas en el Proyecto Real

// Configuración completa de AG Grid
const gridOptions = {
  columnDefs: [
    { 
      field: 'id', 
      headerName: 'ID', 
      width: 80,
      sortable: true, 
      filter: 'agNumberColumnFilter'
    },
    { 
      field: 'name', 
      headerName: 'Nombre', 
      width: 200,
      sortable: true, 
      filter: 'agTextColumnFilter',
      cellRenderer: (params) => {
        return `${params.value}`;
      }
    },
    { 
      field: 'email', 
      headerName: 'Email', 
      width: 250,
      sortable: true, 
      filter: 'agTextColumnFilter'
    },
    { 
      field: 'created_at', 
      headerName: 'Fecha', 
      width: 150,
      sortable: true, 
      filter: 'agDateColumnFilter',
      valueFormatter: (params) => {
        return new Date(params.value).toLocaleDateString();
      }
    }
  ],
  rowData: [],
  pagination: true,
  paginationPageSize: 20,
  paginationPageSizeSelector: [10, 20, 50, 100],
  domLayout: 'autoHeight',
  defaultColDef: {
    resizable: true,
    sortable: true,
    filter: true,
    floatingFilter: true
  },
  onGridReady: (params) => {
    loadData();
  },
  onRowClicked: (params) => {
    console.log('Fila clickeada:', params.data);
  }
};

// Cargar datos
async function loadData() {
  try {
    const response = await fetch('/api/users');
    const data = await response.json();
    gridOptions.api.setRowData(data);
  } catch (error) {
    console.error('Error cargando datos:', error);
  }
}

// Inicializar la grilla
document.addEventListener('DOMContentLoaded', () => {
  const gridDiv = document.querySelector('#myGrid');
  new agGrid.Grid(gridDiv, gridOptions);
});

Lecciones Aprendidas

  1. Paginado del cliente: Mejora significativamente el rendimiento
  2. AG Grid: Supera por mucho a DataTables en funcionalidad y rendimiento
  3. Índices SQL: Son cruciales para consultas rápidas
  4. Eager Loading: Evita el problema N+1
  5. Optimización de consultas: Siempre selecciona solo las columnas necesarias

Conclusión

La combinación de paginado del lado del cliente con AG Grid ha transformado la experiencia de usuario en mis aplicaciones. El rendimiento mejoró drásticamente y la funcionalidad es mucho más rica que con DataTables. AG Grid es definitivamente la mejor opción para tablas de datos complejas y de alto rendimiento.
